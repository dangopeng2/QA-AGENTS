# Account - Ledger 设备识别与匹配规则

> 模块：Account  
> 来源：OK-53772 需求描述整理  
> App 版本：待定  
> 日期：2026-05-18  
> 测试端：Desktop（USB / BLE）

---

## 1. 需求背景

Ledger 连接链路涉及设备识别与匹配、多设备/多账户一致性校验以及账户交互验证。当前业务目标是：
- 在不引入“手动指定设备”UI 的前提下，保证连接行为可预期。
- 在 USB 与 BLE 两条链路上，明确 wallet 聚合与设备匹配规则。
- 降低“同设备重复添加”“多设备助记词不一致”带来的误操作风险。

---

## 2. 功能描述

### 2.1 连接方式与设备搜索

| 项目 | 说明 |
|-----|------|
| 支持方式 | USB、BLE |
| USB 搜索 | 不限制设备型号，只要系统能搜索到即可尝试连接 |
| 设备选择 UI | 当前无“指定连接某一台设备”入口 |
| 多设备行为 | 同时插入多台设备时，连接列表可展示多设备；无论选择哪个连接项，最终均与第一识别设备进行通讯（首设备优先） |

### 2.2 识别顺序

| 项目 | 说明 |
|-----|------|
| 识别顺序规则 | 按系统识别顺序处理，识别到的第一台设备作为实际通讯目标 |
| 稳定性要求 | 同一连接会话内，目标设备不应在无提示下自动切换 |

### 2.3 地址匹配与设备不匹配提示

| 项目 | 说明 |
|-----|------|
| 匹配结果 | 地址匹配通过才可继续业务 |
| 不匹配处理 | 地址不匹配时提示“设备不匹配” |
| 适用范围 | 添加账户、基于已存在网络地址继续添加账户等场景 |

## 3. USB 业务规则

### 3.1 Wallet 添加规则（USB）

| 场景 | 规则 |
|-----|------|
| 同一台设备重复添加 wallet | 允许重复添加，生成新的 wallet（无设备唯一 ID 限制） |
| 多台设备并存 | 只连接识别到的第一台设备 |

### 3.2 Account 添加规则（USB）

| 场景 | 规则 |
|-----|------|
| 当前网络已有地址 | 以“助记词层级”进行校验；不同助记词设备添加时提示“设备不匹配” |
| 当前网络无任何地址 | 不做助记词一致性限制，可正常添加 |
| 同助记词不同设备 | 允许添加（可通过助记词层级校验） |

### 3.3 网络首地址 Hash 约束（USB）

| 项目 | 说明 |
|-----|------|
| 约束锚点 | 某网络首次添加时，记录该网络首地址 Hash 作为后续一致性锚点 |
| 后续校验 | 同网络再次添加时，对比首地址 Hash，不一致则提示“设备不匹配” |
| 跨网络影响 | 未添加过的网络不受已添加网络 Hash 约束 |

示例：
- 设备 A 已添加 BTC，但未添加任何 EVM 网络。
- 切换到不同助记词设备 B 时，仍可在该 wallet 下新增 EVM 账户（因 EVM 网络尚无首地址锚点）。

---

## 4. BLE 业务规则

### 4.1 BLE 唯一标识与聚合

| 项目 | 说明 |
|-----|------|
| 设备标识 | 使用设备蓝牙名称作为唯一标识 |
| 改名影响 | 业务上按蓝牙标识处理，与设备展示名变化解耦 |
| 处理顺序 | 添加 wallet 时先做蓝牙标识匹配，蓝牙匹配通过后再做地址匹配 |

### 4.2 BLE 地址匹配分支

| 场景 | 规则 |
|-----|------|
| 蓝牙匹配通过且地址匹配通过 | 合并到已有 wallet |
| 蓝牙匹配不通过 | 创建新的 wallet |
| 蓝牙匹配通过但地址匹配不通过 | 创建新的 wallet |

---

## 5. 账户交互验证与回归范围

| 类别 | 回归重点 |
|-----|---------|
| 四链交互 | BTC / EVM / SOL / TRON 四条链下，创建地址、签名、转账流程可达 |
| 链路回归 | 添加 wallet / 添加 account 流程在 USB 与 BLE 下行为稳定 |
| 一致性 | USB 与 BLE 在“不匹配提示”和“聚合/新建”分支上的行为一致性 |

---

## 6. 验收标准（Definition of Done）

1. 多设备 USB 并存时，只连接第一识别设备且行为稳定。  
2. USB 在“有首地址锚点网络”下能正确拦截不同助记词设备并提示“设备不匹配”。  
3. USB 在“无首地址锚点网络”下允许新增账户。  
4. BLE 在“蓝牙+地址都匹配”时聚合，在“蓝牙不匹配”或“蓝牙匹配但地址不匹配”时新建 wallet。  
5. BTC / EVM / SOL / TRON 四链创建地址、签名、转账主流程无功能回退。
