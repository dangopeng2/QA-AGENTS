# Perps - 现货交易

> 规则文档：`docs/qa/rules/perps-rules.md` 第 5 章
> 需求文档：`docs/qa/requirements/Perps-现货交易.md`
> 来源 PR：[OneKeyHQ/app-monorepo#11183](https://github.com/OneKeyHQ/app-monorepo/pull/11183)
> 测试端：iOS / Android / 桌面端 / 浏览器插件 / Web
> 变更说明：首版——Hyperliquid 现货交易接入，作为 Perps 模块新增交易模式

---

## 前置条件与测试数据

1. **账户**：已连接 HD 钱包 / HW 钱包 / 私钥钱包 / Keyless Wallet 中任一种，且该钱包已**创建 EVM 地址**（Perps/现货入口依赖 EVM 账户，不区分具体网络）
2. **资产**：账户在 Hyperliquid 有 USDC 余额（合约）且至少持有一种现货代币（含现货 USDC / 非稳定币）
3. **客户端**：覆盖桌面端（TestFlight / MAS / 浏览器插件）、移动端（iOS / Android）、Web
4. **现货元数据已加载**：首次进入时自动调用 `refreshSpotMeta()` 兜底

**范围说明**：现货模式下合约特有功能（触发单 / TP/SL 附单 / 杠杆控件 / 爆仓价 / 持仓线）不支持。

**接口自动化覆盖**：`placeSpotOrder` 服务端参数（`orderType` / `tif` / `assetId` / `isBuy` / `p` / `r`）已由接口自动化覆盖；UI 用例只验证下单后的余额 / 持仓变化与挂单展示。
