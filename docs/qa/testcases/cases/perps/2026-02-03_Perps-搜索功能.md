# Perps - 搜索功能

> 规则引用：`docs/qa/rules/perps-rules.md`

## 测试范围说明

- **支持端**：iOS / Android / Desktop / Extension / Web（全端）
- **范围**：Ticker 模糊搜索（多语言）、Token 选择器版块、中文标签（仅中文环境）、后台配置（分类 tab、代币归属、多语言标签）

## 1. 模糊搜索（多语言 Ticker）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | Token 选择器已打开 | 在搜索框输入英文 Ticker 部分字符（如「BTC」） | 1. 搜索结果列表显示包含该字符匹配的代币<br>2. 列表仅包含匹配项或按服务端规则排序 |
| ❗️❗️P0❗️❗️ | Token 选择器已打开 | 在搜索框输入中文关键词（若服务端已配置中文） | 1. 搜索结果列表显示匹配的代币<br>2. 匹配规则与服务端配置一致 |
| ❗️❗️P0❗️❗️ | Token 选择器已打开，界面语言为英文或日文 | 在搜索框输入中文关键词（若服务端已配置中文） | 1. 搜索结果列表显示匹配的代币<br>2. 列表包含该中文关键词对应的代币 |
| ❗️❗️P0❗️❗️ | Token 选择器已打开 | 在搜索框输入日文 Ticker 或日文关键词（若服务端已配置） | 1. 搜索结果列表显示匹配的代币<br>2. 匹配规则与服务端配置一致 |
| ❗️❗️P0❗️❗️ | Token 选择器已打开 | 输入完整 Ticker（如「ETHUSDC」） | 1. 搜索结果包含该 Ticker 或唯一命中<br>2. 可点击选择该代币 |
| P1 | Token 选择器已打开 | 搜索框输入为空或清空输入 | 1. 显示默认列表或当前分类下的代币<br>2. 不显示「未找到」空状态（或按产品约定显示占位） |
| P1 | Token 选择器已打开 | 输入不存在于服务端数据的关键词（如「XYZZZ999」） | 1. 显示空状态或「未找到相关结果」类提示<br>2. 搜索框可继续输入 |
| P1 | Token 选择器已打开 | 输入超长字符串（如 200 字符） | 1. 前端截断或服务端无结果<br>2. 不崩溃、不报错，展示截断后结果或空状态 |
| P2 | Token 选择器已打开 | 输入特殊字符（如 `<`、`'`、空格） | 1. 按服务端规则返回结果或空状态<br>2. 无 XSS/注入导致页面异常 |

## 2. Token 选择器版块（贵金属、股票、Pre-IPO）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | Token 选择器已打开，服务端已配置版块 | 查看 Token 选择器内的分类/版块区域 | 1. 显示服务端下发的版块（如贵金属、股票、Pre-IPO 等）<br>2. 版块以标签或分组形式展示<br>3. 版块名称与配置一致 |
| ❗️❗️P0❗️❗️ | Token 选择器已打开，版块列表已展示 | 点击「贵金属」版块（若已配置） | 1. 列表切换为该版块下的代币<br>2. 列表仅包含该版块配置的代币或为空（若该版块无代币） |
| ❗️❗️P0❗️❗️ | Token 选择器已打开，版块列表已展示 | 点击「股票」版块（若已配置） | 1. 列表切换为该版块下的代币<br>2. 列表内容与所选版块一致 |
| ❗️❗️P0❗️❗️ | Token 选择器已打开，版块列表已展示 | 点击「Pre-IPO」版块（若已配置） | 1. 列表切换为该版块下的代币<br>2. 列表内容与所选版块一致 |
| P1 | Token 选择器已打开 | 服务端未配置某版块或下架版块 | 1. 该版块不显示在 Token 选择器内<br>2. 仅显示服务端当前下发的版块 |
| P1 | 已选择某版块并展示列表 | 切换到另一版块，再切回原版块 | 1. 列表展示与版块一致<br>2. 无错乱或残留上一版块数据 |
| P2 | 某版块下无代币（服务端配置为空） | 点击该版块 | 1. 显示空状态或「暂无代币」类提示<br>2. 不显示其他版块代币 |

| ❗️❗️P0❗️❗️ | 界面语言为中文，行情页顶部有带中文数据的已收藏代币 | 进入行情页，查看顶部代币列表 | 1. 代币项显示中文标签（若有）<br>2. 展示位置与设计一致 |
| ❗️❗️P0❗️❗️ | 界面语言为英文或日文，列表中有带中文数据的代币 | 打开 Token 选择器或进入行情页，查看代币列表项 | 1. 不显示中文标签<br>2. 仅显示主 Ticker 或默认名称 |

## 4. 搜索与版块组合、服务端配置

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 已选择某版块（如贵金属） | 在该版块下输入搜索关键词 | 1. 搜索结果仅来自当前版块下的代币<br>2. 结果与关键词匹配且属于该版块（或为空） |

| P2 | 弱网或接口超时 | 打开 Token 选择器、切换版块或输入搜索 | 1. 显示加载状态或超时提示<br>2. 不崩溃，可重试或重新操作 |

## 5. 后台配置（分类 Tab、代币归属、多语言标签）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 有后台配置权限 | 在后台新增一个分类 tab（如自定义名称），保存并发布 | 1. 保存成功或显示发布成功<br>2. C 端刷新 Token 选择器后，分类区域显示该新分类 tab<br>3. 新分类 tab 名称与配置一致 |
| ❗️❗️P0❗️❗️ | 有后台配置权限，已存在多个分类 | 在后台将代币 A 配置到某分类（如「贵金属」），保存并发布 | 1. 保存成功或显示发布成功<br>2. C 端选择该分类 tab 后，列表包含代币 A<br>3. 代币 A 不在未配置该代币的其他分类下列出（或按产品规则：多分类则多处显示） |
| ❗️❗️P0❗️❗️ | 有后台配置权限 | 在后台为某代币配置多语言标签（如中文名、英文名、日文名），保存并发布 | 1. 保存成功或显示发布成功<br>2. C 端界面语言为中文时，该代币显示配置的中文标签<br>3. C 端界面语言为英文/日文时，该代币显示配置的对应语言标签<br>4. 搜索框输入各语言标签关键词，均可搜到该代币 |
| P1 | 有后台配置权限 | 在后台编辑某分类名称或排序，保存并发布 | 1. 保存成功<br>2. C 端 Token 选择器内该分类名称或顺序与配置一致 |
| P1 | 有后台配置权限 | 在后台下架某分类或移除某代币与分类的归属，保存并发布 | 1. 保存成功<br>2. C 端该分类不显示或该分类下不包含该代币（与配置一致） |
| P1 | 后台已配置某代币多语言标签 | C 端切换界面语言（中/英/日）后打开 Token 选择器，查看该代币 | 1. 列表项显示当前界面语言对应的标签<br>2. 与后台配置一致 |
| P2 | 有后台配置权限 | 在后台新增分类但不配置任何代币，保存并发布 | 1. 保存成功<br>2. C 端点击该分类时显示空状态或「暂无代币」类提示 |

## 6. 现货 Tab 搜索

> **说明**：Token 搜索框新增「现货」Tab，与「收藏」「合约」并列。现货 Tab 内的搜索逻辑独立于合约 Tab。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 打开 Token 搜索框 → 切到「现货」Tab | 在搜索框输入 `BTC` | 1. 匹配 UBTC（显示名 BTC）的现货交易对<br>2. 不区分大小写，输入 `btc` 结果一致<br>3. 仅显示现货交易对，不混合合约代币 |
| ❗️❗️P0❗️❗️ | 现货 Tab 内搜索 | 输入交易对格式 `BTC/USDC` | 1. 命中对应行<br>2. pair 格式按 `<base>/<quote>` 模糊匹配 |
| ❗️❗️P0❗️❗️ | 现货 Tab 内搜索 | 输入原始内部名 `UBTC`（未做映射的原始 Ticker） | 1. 同样命中 BTC 显示名对应的代币<br>2. 显示名与原始名均可搜索 |
| ❗️❗️P0❗️❗️ | 现货 Tab 列表展示 | 不输入搜索，查看默认列表 | 1. 列表按默认字段「24h 成交量 `dayNtlVlm`」降序<br>2. `dayNtlVlm < 10` 的代币不显示（最小名义成交额过滤）<br>3. 列头：资产 / 标记价 / 24h 涨跌 / 24h 成交量 / 市值（无资金费率列） |
| P1 | 现货 Tab 内空搜索 | 输入不存在的代币（如 `ZZZZZZ`） | 1. 显示空状态文案<br>2. 搜索框可继续输入 |
| P1 | 现货 / 合约 Tab 切换搜索 | 在合约 Tab 输入 `BTC` 后切到现货 Tab | 1. 现货 Tab 不复用合约 Tab 的搜索关键词，或按产品约定保持<br>2. 现货 Tab 列表正确展示现货数据 |
| P2 | 现货 Tab 长列表滚动 | 滚动到列表底部（100+ 代币） | 1. 滚动流畅（FPS ≥ 30）<br>2. 所有过滤后代币正确加载 |
