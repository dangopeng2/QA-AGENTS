## 前置条件与测试数据

1. 已导入 HD 钱包，并在 Optimism / Base / Arbitrum 三条 L2 链下各自创建账户1、账户2，均持有 ETH 与 OP / USDC / ARB 等对应代币余额
2. 转账方向固定为**账户1 → 账户2**，余额不足时自动反向（账户2 → 账户1）
3. 区块浏览器入口可正常访问（链分别选择 Optimism / Base / Arbitrum）

测试数据：

| # | 链 | 币种 | 金额 | 备注 |
| --- | --- | --- | --- | --- |
| 1 | Optimism | ETH | 最大值 | Max（含 L1 + L2 费用扣减） |
| 2 | Optimism | OP | 0 | 拦截 |
| 3 | Base | ETH | 最大值 | Max（含 L1 + L2 费用扣减） |
| 4 | Base | USDC | 最大值 | ERC20 Max（不扣 Gas） |
| 5 | Arbitrum | ETH | 最大值 | Max（含 L1 + L2 费用扣减） |

## 1. 主币 / 代币转账主流程（参数化）

> 主流程统一为：发送 → 选币 → 输入地址 → 输入金额 → 切法币 → 预览 → 确认 → 历史 → 区块浏览器对账。每行用例聚焦该参数差异点，通用流程预期收敛至首行后续行写「同上」。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 1. 链：Optimism、币种：ETH<br>2. 金额=最大值 | 1. 切换 Optimism 链账户1<br>2. 点击「发送」，进入币种选择页<br>3. 选择代币 ETH<br>4. 输入收款地址=账户2<br>5. 金额点击「Max」自动填入，点击切换法币展示<br>6. 点击「预览」进入交易确认页<br>7. 在确认页核对字段后点击「确认」<br>8. 进入历史记录，打开该笔记录并点击交易哈希跳转区块浏览器 | 1. 输入账户2 地址后显示**账户名标签**<br>2. 法币估值正确，无数量级错误<br>3. 交易确认页显示：**网络** = Optimism、**发送账户地址及标签**（账户1）、**金额与 icon (ETH)**、**收款地址及标签**（账户2）、**进阶设置**入口（Gas Limit / Max Fee / Max Priority Fee / Nonce / Hex Data）、**网络费用四档**（Slow / Standard / Fast / Custom）可切换<br>4. **Max = ETH 余额 − 当前档位预估总费用（含 L1 数据费 + L2 执行费）**；切换档位后 Max 同步刷新<br>5. 点击确认后弹窗关闭，提示「交易已提交」<br>6. 历史记录新增该笔，显示：**金额**、**类型**=转出、**状态**（处理中 / 成功 / 失败）、**时间格式正确**、**from / to**、**交易哈希**、**网络费用**、**网络** = Optimism、**区块高度**、**Nonce 值**、**确认数**<br>7. 点击交易哈希跳转 Optimism 区块浏览器，**金额 / symbol / from / to / 费用 / 区块高度 / Nonce** 与 App 一致 |
| ❗️❗️P0❗️❗️ | 1. 链：Optimism、币种：OP<br>2. 金额=0 | 1. 选择 OP<br>2. 输入账户2 地址<br>3. 金额输入 0 | 1. 显示提示「不能发送 0」类文案<br>2. 「预览」按钮置灰，不可进入下一步 |
| ❗️❗️P0❗️❗️ | 1. 链：Base、币种：ETH<br>2. 金额=最大值 | 同首行（按场景参数，链切换为 Base） | 1. **网络** = Base，区块浏览器跳转 Base Scan<br>2. Max 计算同样含 L1 + L2 费用扣减<br>3. 其余预期同首行 |
| ❗️❗️P0❗️❗️ | 1. 链：Base、币种：USDC<br>2. 金额=最大值 | 1. 切到 Base，选择 USDC<br>2. 输入账户2 地址<br>3. 金额点击「Max」<br>4. 完成预览 → 确认 → 历史 → 区块浏览器 | 1. **Max = USDC 全部余额**（代币 Max 不扣 Gas，因 Gas 由主币 ETH 支付）<br>2. ETH 余额需足够支付 L1 + L2 总费用，否则提示资金不足<br>3. 交易确认页 icon / symbol = USDC；网络费用以 **ETH** 结算<br>4. 其余预期同首行 |
| ❗️❗️P0❗️❗️ | 1. 链：Arbitrum、币种：ETH<br>2. 金额=最大值 | 同首行（按场景参数，链切换为 Arbitrum） | 1. **网络** = Arbitrum，区块浏览器跳转 Arbiscan<br>2. Max 计算同样含 L1 + L2 费用扣减<br>3. 其余预期同首行 |
| ❗️❗️P0❗️❗️ | 1. 链：Arbitrum、币种：ARB<br>2. 金额=中间值 | 1. 切到 Arbitrum，选择 ARB<br>2. 输入账户2 地址<br>3. 输入中间值金额<br>4. 完成预览 → 确认 → 历史 → 区块浏览器 | 1. 交易确认页 icon / symbol = ARB；网络费用以 **ETH** 结算<br>2. 其余预期同首行 |

## 2. 网络费用四档与进阶设置（EIP-1559 + L2）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 四档切换 | 1. 进入任一 L2 链转账确认页<br>2. 依次切换 Slow / Standard / Fast / Custom | 1. 每档显示**预估费用**（含 L1 数据费 + L2 执行费）与**预估到账时间**，二者随档位联动<br>2. 切换后 Max 金额（如使用 Max）同步刷新 |
| ❗️❗️P0❗️❗️ | 进阶设置可修改（1559 字段） | 1. 在确认页打开「进阶设置」<br>2. 查看并尝试修改 Gas Limit / **Max Fee Per Gas** / **Max Priority Fee Per Gas** / Nonce / Hex Data | 1. 五项均可见且可编辑（L2 均支持 EIP-1559）<br>2. 修改后预估费用同步刷新<br>3. 非法值（负数 / 非数字 / Priority Fee > Max Fee 等）有校验提示 |
| P1 | L1 数据费随主网波动 | 1. 在 Ethereum 主网 Gas 高峰期与低谷期分别打开 L2 转账确认页 | 1. 高峰期总费用显著高于低谷期<br>2. 费用差异主要来自 L1 数据费部分（如 App 提供 breakdown 可对照） |
| P1 | Custom 档位生效 | 1. 切到 Custom，自定义 Max Fee 高于 Standard<br>2. 预览 → 确认 → 广播 | 1. 历史记录与区块浏览器中网络费用 ≈ 实际有效 Gas Price × Gas Used + L1 数据费<br>2. 上链速度符合自定义档位预期 |
| P1 | Nonce 顺序广播 | 1. 修改进阶设置 Nonce 为当前账户的下一个空缺值<br>2. 广播 | 历史记录与区块浏览器中 Nonce 与设置值一致 |

## 3. 性能与体验

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| P1 | 主要 loading 时长 | 完成一笔小额 ETH 转账（任一 L2 链）：从点击发送到预览页可交互；从确认到历史记录新增 | 单次主要等待 < 2s（弱网另测） |
| P2 | 重复进入发送页 | 同一 L2 链下，连续 5 次打开「发送」再返回 | 无卡顿、无内存异常；列表与余额展示无错乱 |
| P2 | 多 L2 链切换 | 在 Optimism / Base / Arbitrum 之间快速切换 5 次 | 余额、代币列表、历史记录与所选链一致，无串台 |
