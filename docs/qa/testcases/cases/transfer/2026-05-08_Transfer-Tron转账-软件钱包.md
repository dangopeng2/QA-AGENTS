## 前置条件与测试数据

1. 已导入 HD 钱包，并在 Tron 主网下创建账户1、账户2，均持有 TRX 与 USDT-TRC20 余额
2. 转账方向固定为**账户1 → 账户2**，余额不足时自动反向（账户2 → 账户1）
3. 区块浏览器入口可正常访问（链选择 Tron）

## 1. 主币 / 代币转账主流程（参数化）

> 主流程统一为：发送 → 选币 → 输入地址 → 输入金额 → 切法币 → 预览 → 确认 → 历史 → 区块浏览器对账。每行用例聚焦该参数差异点，通用流程预期收敛至首行后续行写「同上」。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 1. 代币：TRX<br>2. 金额=0.001 | 1. 切换 Tron 网络下账户1<br>2. 点击「发送」，进入币种选择页<br>3. 选择代币 TRX<br>4. 输入收款地址=账户2<br>5. 输入金额 0.001，点击切换法币展示<br>6. 点击「预览」进入交易确认页<br>7. 在确认页核对字段后点击「确认」<br>8. 进入历史记录，打开该笔记录并点击交易哈希跳转区块浏览器 | 1. 输入账户2 地址后显示**账户名标签**<br>2. 法币估值正确，无数量级错误<br>3. 交易确认页显示：**网络** = Tron、**发送账户地址及标签**（账户1）、**金额与 icon (TRX)**、**收款地址及标签**（账户2）、**能量带宽抵扣数据**（仅扣带宽）、**网络费用** = 固定档位预估值<br>4. 点击确认后弹窗关闭，提示「交易已提交」<br>5. 历史记录新增该笔，显示：**金额**、**类型**=转出、**状态**（处理中 / 成功 / 失败）、**时间格式正确**、**from / to**、**资源**（扣除的能量 + 带宽）、**交易哈希**、**网络费用**、**网络** = Tron、**区块高度**、**确认数**<br>6. 点击交易哈希跳转区块浏览器，**金额 / symbol / from / to / 费用 / 区块高度** 与 App 一致 |
| ❗️❗️P0❗️❗️ | 1. 代币：TRX<br>2. 金额=Max | 1. 同首行<br>2. 金额点击「Max」自动填入<br>3. 完成预览 → 确认 → 历史 → 区块浏览器 | 1. **Max = 余额 − 网络费用预估**，发送后账户1 余额接近 0<br>2. 其余预期同上 |
| ❗️❗️P0❗️❗️ | 1. 代币：TRX<br>2. 金额=0 | 1. 选 TRX<br>2. 输入账户2 地址<br>3. 金额输入 0 | 1. 显示提示「不能发送 0」类文案<br>2. 「预览」按钮置灰，不可进入下一步 |
| ❗️❗️P0❗️❗️ | 1. 代币：USDT (TRC20)<br>2. 金额=0.0002 | 同首行（按场景参数） | 1. 同首行 1 / 2 / 4 / 6<br>2. 交易确认页 icon / symbol = USDT；网络费用 = 固定档位<br>3. **能量带宽抵扣数据**：同时显示**能量**与**带宽**消耗（USDT 为 TRC20 合约调用）<br>4. 历史记录的**资源**字段同时显示能量 + 带宽数值 |

## 2. 资源不足烧 TRX 边界

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 能量与带宽均不足，发起 USDT 转账 | 1. 在能量与带宽均不足的账户上发起 USDT 转账<br>2. 查看交易确认页费用展示<br>3. 广播后查看历史记录与区块浏览器 | 1. 确认页**显示烧 TRX 等价费用**（如「需消耗 X TRX 抵扣能量」类文案）<br>2. 历史记录与区块浏览器中网络费用对得上 |
