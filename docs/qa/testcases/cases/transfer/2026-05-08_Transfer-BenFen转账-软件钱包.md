## 前置条件与测试数据

1. 已导入 HD 钱包，并在 BenFen 主网下创建账户1、账户2，均持有 BFC、BUSD、BF_USDC、LONG 余额
2. 转账方向固定为**账户1 → 账户2**，余额不足时自动反向（账户2 → 账户1）
3. 区块浏览器入口可正常访问（链选择 BenFen）

## 1. 主币 / 代币转账主流程（参数化）

> 主流程统一为：发送 → 选币 → 输入地址（账户2）→ 输入金额 → 切法币 → 预览 → 确认 → 历史 → 区块浏览器对账。每行用例聚焦该参数差异点，通用流程预期收敛至首行后续行写「同上」。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 1. 币种：BFC<br>2. 金额=0.000000001（最小单位） | 1. 进入钱包首页，切换 BenFen 链下账户1<br>2. 点击「发送」，进入币种选择页<br>3. 选择代币 BFC<br>4. 输入收款地址=账户2<br>5. 输入金额 0.000000001，点击切换法币展示<br>6. 点击「预览」进入交易确认页<br>7. 在确认页核对字段后点击「确认」<br>8. 进入历史记录，打开该笔记录并点击交易哈希跳转区块浏览器 | 1. 输入账户2 地址后显示**账户名标签**<br>2. 金额输入框正常显示 **0.000000001 BFC**，无科学计数异常或截断；法币估值正确无数量级错误<br>3. 交易确认页显示：**网络** = BenFen、**发送账户地址及标签**（账户1）、**金额与 icon (BFC)**、**收款地址及标签**（账户2）、**网络费用**（以 BFC 结算）<br>4. 点击确认后弹窗关闭，提示「交易已提交」<br>5. 历史记录新增该笔，显示：**金额**、**类型**=转出、**状态**（处理中 / 成功 / 失败）、**时间格式正确**、**from / to**、**交易哈希**、**网络费用**、**网络** = BenFen<br>6. 点击交易哈希跳转 BenFen 区块浏览器，**金额 / symbol / from / to / 费用** 与 App 一致 |
| ❗️❗️P0❗️❗️ | 1. 币种：BFC<br>2. 金额=Max | 1. 同首行<br>2. 金额点击「Max」自动填入<br>3. 完成预览 → 确认 → 历史 → 区块浏览器 | 1. **Max = 余额 − 当前预估网络费用**；广播后账户1 余额接近 0（或保留账户最低余额，按链规则）<br>2. 其余预期同首行 |
| ❗️❗️P0❗️❗️ | 1. 币种：BFC<br>2. 金额=0 | 1. 选择 BFC<br>2. 输入账户2 地址<br>3. 金额输入 0 | 1. 显示提示「不能发送 0」类文案<br>2. 「预览」按钮置灰，不可进入下一步 |
| ❗️❗️P0❗️❗️ | 1. 币种：BUSD<br>2. 金额=0.000000001 | 1. 切换至 BUSD<br>2. 输入账户2 地址<br>3. 输入金额 0.000000001<br>4. 完成预览 → 确认 → 历史 → 区块浏览器 | 1. 交易确认页 icon / symbol = BUSD；网络费用以主币 **BFC** 结算，与代币余额分离展示<br>2. 极小精度金额输入与展示无截断、无科学计数异常<br>3. 其余预期同首行 |
| ❗️❗️P0❗️❗️ | 1. 币种：LONG<br>2. 金额=0 | 同 BFC 0 行（按场景参数） | 同上 |
| ❗️❗️P0❗️❗️ | 1. 币种：BF_USDC<br>2. 金额=Max | 1. 切换至 BF_USDC<br>2. 输入账户2 地址<br>3. 金额点击「Max」自动填入 BF_USDC 余额<br>4. 完成预览 → 确认 → 历史 → 区块浏览器 | 1. **Max = BF_USDC 全部余额**（代币 Max 不扣 Gas，因 Gas 由主币 BFC 支付）<br>2. 主币 BFC 余额需足够支付 Gas，否则提示资金不足<br>3. 交易确认页 icon / symbol = BF_USDC；网络费用以 **BFC** 结算<br>4. 其余预期同首行 |
