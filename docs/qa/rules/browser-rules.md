# Browser 模块测试规则文档

> 本文档记录 Browser（DApp）模块的核心测试规则。
> 生成 Browser/DApp 相关测试用例时，必须参考本文档。

---

## 1. 模块概述

Browser 模块包含以下核心功能：
- DApp 浏览器
- 钱包连接（WalletConnect、Injected）
- 签名请求处理
- 交易确认
- 权限管理

---

## 2. DApp 连接规则

### 2.1 连接方式

| 连接方式 | 说明 | 适用场景 |
|---------|-----|---------|
| Injected | 注入式连接 | Extension、Desktop |
| WalletConnect | 扫码连接 | Mobile、跨设备 |
| EIP-6963 | 多钱包发现 | Extension |

### 2.2 连接状态机

```
Disconnected → Connecting → Connected → Disconnected
                  ↓
              Failed (超时/拒绝)
```

### 2.3 连接规则

| 规则项 | 规则描述 |
|-------|---------|
| Origin 展示 | 显示 DApp 域名 |
| 权限请求 | 显示请求的权限范围 |
| 账户选择 | 支持选择连接的账户 |
| 网络选择 | 支持选择连接的网络 |

## 3. 签名请求规则

### 3.1 签名类型

| 签名类型 | 说明 | 风险等级 |
|---------|-----|---------|
| personal_sign | 个人签名 | 低 |
| eth_signTypedData | 类型化数据签名 | 中 |
| eth_sign | 原始签名 | 高（危险） |
| eth_sendTransaction | 交易签名 | 高 |

### 3.2 签名内容解析

| 规则项 | 规则描述 |
|-------|---------|
| 内容解析 | 尽可能解析签名内容 |
| 可读展示 | 以可读格式展示（非十六进制） |
| 风险提示 | 高风险操作显示警告 |

### 3.3 盲签处理

| 场景 | 规则 |
|-----|------|
| 解析成功 | 显示解析后的内容 |
| 解析失败 | 显示"无法解析"警告 |
| 盲签阻断 | 高危操作强制警告，需二次确认 |

## 8. 通用规则

### 状态机
- connect → 请求权限 → 签名/交易 → 用户确认/拒绝 → 回到 DApp 回传

### 安全
- origin 展示
- 权限范围
- 钓鱼/相似域名
- 盲签阻断（解析失败时强警示）

### 兼容
- 多 tab
- EIP-6963
- 多钱包冲突
- 网络切换请求处理

### 链上不确定性
- 签名弹窗中断
- 返回后恢复
- 重复请求幂等

### 可观测
- 签名内容解析结果可验证
- method/params/金额/收款地址/合约地址

## 8.1 账户对齐与同步模式（首要账户）

本章节用于约束 `设置 - 对齐首要账户` 与 `设置 - 账户派生路径` 共同作用下的 DApp 连接同步逻辑。

### 8.1.1 配置项与适用范围

| 配置项 | 取值 | 描述 |
|---|---|---|
| 对齐首要账户 | 将 dApp 账户对齐至钱包（默认） | 钱包侧在满足条件时同步 DApp 标签页的账户/网络 |
| 对齐首要账户 | 独立模式 | 钱包侧与 DApp 标签页账户/网络完全独立 |
| 对齐首要账户 | 始终使用钱包账户 | DApp 的账户随钱包首页账户切换联动，但网络保持独立 |
| 账户派生路径 | BIP44 <-> Ledger Live | 断开并重新连接后，DApp 侧显示地址需随派生路径更新 |

### 8.1.2 将 dApp 账户对齐至钱包（默认）

- 单一连接同步：当某个 DApp 标签页仅连接单一钱包账户与单一网络时，钱包侧显示的账户与网络需要与该 DApp 连接状态保持一致。
- 多连接不同步：当某个 DApp 标签页同时连接多个钱包账户与网络时，钱包侧不需要同步 DApp 的多连接状态。
- 多标签隔离：在不同 DApp 标签页切换其连接账户/网络时，不应影响其他已连接标签页的网络展示状态。
- 派生路径一致性：切换 `账户派生路径`（BIP44 <-> Ledger Live）后，断开并重新连接；钱包显示的派生地址与 DApp 标签页显示的派生地址需要一致。
- 交易确认校验：在构建交易并进入交易确认页时，From/相关地址展示需对应当前派生路径下的地址。

### 8.1.3 独立模式

- 钱包独立：钱包首页的账户/网络切换不影响 DApp 标签页的连接账户/网络。
- DApp 独立：DApp 标签页的连接账户/网络切换不影响钱包首页的账户/网络展示。
- 多标签隔离：不同 DApp 标签页之间的连接状态不互相干扰。

### 8.1.4 始终使用钱包账户

- 账户联动：钱包首页账户切换后，DApp 标签页的连接账户需要随之更新。
- 网络独立：DApp 标签页的连接网络不随钱包网络自动同步变更。
- 未创建地址：当切换到未创建地址对应的账户时，不触发 DApp 侧账户同步；地址创建完成后触发联动并可连接到 DApp。
- 私钥/观察地址系列约束：
  - 同系列：私钥账户与观察地址之间切换联动生效。
  - 不同系列：不应触发账户联动。
- 连接管理入口约束：在始终使用钱包账户模式下，DApp 连接管理中不应出现网络与账户的切换入口。

### 8.1.5 异常与鲁棒性

- 断开并重新连接：需要恢复对应模式的同步逻辑。
- 切换过程可中断：在切换过程中模拟中断（关闭钱包、切换到其他应用等）后，不应产生跨账户/跨网络错乱状态。
- 快速切换：快速切换账户与网络时，不应出现同步混乱或交互异常。
- 模式切换一致性：在三种模式间任意切换后，需要符合当前启用模式的同步逻辑。

## 8.2 DApp 翻译设置入口规则

本章节用于约束 DApp 浏览器翻译入口与翻译设置弹层/面板的展示与行为一致性。

### 8.2.1 入口位置与触发

| 入口场景 | 规则 |
|---------|------|
| Desktop：地址栏左侧工具区 | 展示翻译图标与提示文案，点击后在 Popover 中展示翻译设置内容 |
| iOS：底部栏翻译按钮 | 底栏出现翻译入口，点击后弹出翻译设置面板 |
| Android：底部栏翻译按钮 | 底栏新增翻译按钮且支持手势点击，点击后打开翻译设置弹层 |

### 8.2.2 设置面板内容

| 面板内容 | 规则 |
|---------|------|
| 引擎、目标语言、显示模式选项 | 设置弹层/面板中展示并可用于配置翻译 |
| 可执行翻译动作 | 在设置弹层/面板内触发翻译动作 |

### 8.2.3 翻译状态切换与按钮同步（Android）

| 场景 | 规则 |
|------|------|
| 翻译状态切换 | 底栏翻译按钮图标应同步变化（未翻译 / 已翻译） |

### 8.2.4 翻译配置持久化（Jotai）

- 用户修改引擎/目标语言/显示模式后，重新进入浏览器时翻译配置按上次选择保留，不回退到默认值

### 8.2.5 功能开关一致性（PrimeFeatures / DAppTranslate）

- DAppTranslate 枚举接入功能开关（PrimeFeatures）后，翻译相关入口在开关控制场景下保持一致行为

## 8.3 侧边栏 Tab 排序偏好设置规则

本章节用于约束 Browser 竖向侧边栏中新建 Tab 的插入位置偏好、滚动行为与相关边界逻辑。

### 8.3.1 偏好项与入口

| 配置项 / 场景 | 规则 |
|--------------|------|
| 持久化偏好 `newBrowserTabPosition` | 仅支持 `top` 与 `bottom` 两个取值，默认值为 `bottom` |
| 侧边栏展开态 | 底部右下角显示 Settings 图标，点击后弹出 ActionList |
| 侧边栏收起态 | 不显示 Settings 图标 |
| ActionList 菜单 | 提供 `新标签位置` 选项，包含 `顶部` 与 `底部` |

### 8.3.2 新建 Tab 插入与自动滚动

| 模式 | 规则 |
|------|------|
| `top` | 新建 Tab 插入到 unpinned 区顶部，侧边栏自动滚动到顶部 |
| `bottom` | 新建 Tab 追加到 unpinned 区底部，侧边栏自动滚动到底部 |
| 连续新建 | 按当前偏好持续叠放，最新创建的 Tab 位于当前偏好对应的一端 |
| 偏好切换 | 仅影响后续新建 Tab，不主动重排已有 unpinned Tab |

### 8.3.3 URL 变化与排序保持

- `top` 模式下，Tab 创建后在页内导航或 URL 变化时保持创建时位置，不因 `setWebTabData` 重置 `timestamp` 而跳到底部。
- `bottom` 模式下，URL 变化行为保持旧逻辑。

### 8.3.4 Popover 与侧边栏展开态保护

- Settings 的 ActionList 打开期间，通过 `reportPopoverOpen` 阻止侧边栏因 hover 区域变化被误收起。
- 鼠标移入 ActionList 弹出层时，侧边栏保持展开。
- 关闭 ActionList 后，若鼠标已离开侧边栏与弹出层区域，侧边栏再按原逻辑收起。

### 8.3.5 与 Pinned / 拖拽 / Unpin 的关系

| 场景 | 规则 |
|------|------|
| Pinned Tabs | 始终位于 unpinned 区上方，切换偏好不打乱 Pinned 区顺序 |
| 拖拽排序 | 已通过拖拽形成的现有顺序不因偏好切换被覆盖 |
| `top` 模式下 Unpin | Unpin 后的 Tab 回到 unpinned 区顶部 |

### 8.3.6 持久化与兼容性

- 重启 App 后保留上次选择的 `新标签位置` 偏好。
- 老用户升级后默认值仍为 `bottom`，保持原有新增 Tab 追加到底部的行为。
